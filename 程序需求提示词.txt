作为一个资深的 Python 量化交易开发者，请帮我编写一个基于 ccxt 库的实时加密货币监控脚本。
1. 核心目标：
监控 Binance 交易所所有 USDT 交易对（现货或合约均可，优先合约），寻找符合“威科夫量价异常（Effort vs Result）”形态的交易机会。
2. 策略逻辑（核心算法）：
对于每一个交易对，实时监控刚刚收盘的 1分钟 K线（ohlcv）：
条件 A（巨量）： 该 1分钟 K线的成交量（Volume）必须大于该币种过去 24小时（1440分钟）内所有 K线的成交量。
条件 B（窄幅）： 该 1分钟 K线的振幅（(High - Low) / Open）必须小于 0.5%（请将此参数设为变量 AMPLITUDE_THRESHOLD，方便调整）。
条件 C（基本过滤）： 为了防止无效报错，过滤掉 24小时成交额低于 50 万 USDT 的极冷门币种（设为变量 MIN_24H_VOLUME_USDT）。
3. 技术要求：
异步架构： 由于需要监控数百个币种，必须使用 asyncio 和 ccxt.async_support 来并发处理请求，极大化运行效率。
API限流处理： 请内置智能的限流机制（Rate Limiting），防止因请求过快被交易所封禁 IP。
本地缓存（优化点）： 为了避免每分钟都请求 1440 根历史 K线，请设计一个简单的本地缓存机制，只在初始化时拉取历史数据，之后每分钟实时更新最新 K线并维护一个定长队列（Deque）。
4. 报警推送：
当检测到满足条件的币种时，通过 HTTP Request 发送消息到钉钉（DingTalk）或飞书（Lark）的 Webhook。
消息内容需包含：币种名称、发生时间、当前价格、振幅百分比、成交量倍数（是过去24H平均量的几倍）。
5. 输出要求：
代码需要有详细的中文注释。
请将配置项（API Key, Webhook URL, 阈值参数）集中在代码顶部的 Config 区域。
代码具有极强的鲁棒性，网络超时或报错后可以自动重试，不会崩溃。
6.网络环境：大陆用户；或者代理成美国IP。
